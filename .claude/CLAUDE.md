# 项目规则

## 核心目录

| 目录 | 说明 |
|------|------|
| `_posts/` | 已发布文章 |
| `_drafts/` | 草稿，不会被 Jekyll 构建 |
| `assets/img/posts/` | 文章配图 |

---

## 禁止用脚本做批量修改替换

对本项目里的文件做任何修改，禁止编写 Python（或其他脚本语言）脚本进行批量查找替换，也不允许用 `sed`、`awk` 等命令行工具做批量替换。

所有文件修改必须使用 Edit 工具逐处进行，即使是重复性的替换（比如统一替换某个符号、词语）也一样。

---

## 文章分类体系

单级分类，每篇文章只归属一个，共 5 个：

| 分类 | 判断标准 |
|------|---------|
| **AI 原理** | 讲"是什么、为什么"——底层机制、算法原理 |
| **AI Agent** | 讲"Agent 怎么设计"——架构、工程方法 |
| **AI 开发** | 讲"怎么写代码/搭系统"——框架、技术选型 |
| **AI 工具** | 讲"用什么工具/怎么用"——产品、效率工具 |
| **AI 观察** | 讲"行业怎么了"——政策、商业、趋势 |
