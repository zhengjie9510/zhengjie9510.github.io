# Git 安全提交 Skill

## ⚠️ 绝对禁止（除非你获得用户明确同意）

**以下两个命令严禁自动执行：**
- `git commit` 
- `git push`

**违规后果：** 未经用户同意就提交会污染代码历史，可能导致不可挽回的错误。

---

## 执行前自检清单（每次执行前必须默念）

在输入任何 git 命令前，问自己：
- [ ] 我要执行的是 commit 吗？→ **必须询问用户**
- [ ] 我要执行的是 push 吗？→ **必须询问用户**
- [ ] 其他命令（add/status/diff）→ 可以直接执行

---

## ✅ 可以自动执行（无需确认）

- **文件修改**：编辑、格式化、规范化文件内容
- **文件重命名**：按规范重命名文件
- **添加到暂存区**：`git add -A` 或 `git add 特定文件`
- **查看状态**：`git status`、`git diff` 等查看类命令
- **下载/复制资源**：下载图片、复制文件等
- **删除临时文件**：删除 .DS_Store 等系统文件
- **创建目录**：创建所需的文件夹结构

---

## 🛑 必须询问确认（使用 AskUserQuestion 工具）

### 何时询问
**任何情况下**，只要你想执行 commit 或 push，必须先问。

### 如何询问
必须使用 **AskUserQuestion** 工具，提供明确的选项按钮：

#### Commit 前询问示例：
```
用户处理完成后：
"更改已完成，准备提交：
- 修改文件：xxx.md
- 新增文件：yyy.png

是否提交到本地仓库？"
选项：
- ✅ 是，提交（使用提交信息：'xxx'）
- ✏️ 修改提交信息
- ❌ 否，先不提交
```

#### Push 前询问示例：
```
"已提交到本地仓库，提交记录：abc123

是否推送到远程仓库 origin/main？"
选项：
- ✅ 是，推送
- ❌ 否，先不推送
```

## 完整工作流程（必须严格遵守）

### 场景：处理博客文章并提交

```
用户：帮我处理这篇博客并提交

AI 执行：
1. ✅ 读取原文件
2. ✅ 重命名文件
3. ✅ 添加 front matter
4. ✅ 下载图片
5. ✅ git add -A
6. ✅ git status（查看变更）

7. 🛑 【询问】使用 AskUserQuestion：
   "更改已准备就绪：
   - 重命名：xxx → yyy
   - 下载图片：3张
   
   是否提交？"
   
8. 用户选择 ✅ → 执行 git commit
   用户选择 ❌ → 停止，告知已暂存

9. 🛑 【再次询问】使用 AskUserQuestion：
   "是否推送到远程？"
   
10. 用户选择 ✅ → 执行 git push
    用户选择 ❌ → 停止，告知稍后手动推送
```

## 违规检查

如果你发现自己在没有询问的情况下执行了 commit 或 push：
1. 立即向用户道歉
2. 说明违反的规则
3. 回顾提交的内容
4. 询问用户是否需要回滚
